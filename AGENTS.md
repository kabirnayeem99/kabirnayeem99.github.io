# AGENTS.md

Project guidance for AI/code agents working in this repository.

## Core stack rule (mandatory)

- Build and edit this site using raw `HTML` and `CSS`.
- Use `JavaScript` only when clearly necessary for progressive enhancement.
- Do not introduce frameworks or JS build tools (no React, Vue, Angular, Next, Astro, Vite, Webpack, npm pipeline) unless explicitly requested by the user.
- Do not add unnecessary dependencies.
- The existing Python render step is part of the repo. Use it instead of adding another generation pipeline.

## Source of truth

- Site content lives in `content/site-content.json`.
- Rendered pages are generated by the renderer module, invoked through `python3 src/main.py --render-site`.
- Edit CSS in `assets/css/styles.source.css`.
- `assets/css/styles.css` is generated and minified output.
- Root and localized HTML files are generated outputs and should normally be updated through the render script, not hand-edited.

## Project architecture

- This is a static multilingual GitHub Pages site.
- English pages are generated at the root:
  - `index.html`, `work.html`, `project.html`, `blog.html`, `stats.html`
- Localized pages mirror structure in:
  - `bn/`, `ar/`, `ur/`
- Shared assets:
  - `assets/css/styles.source.css`
  - `assets/css/styles.css`
  - `assets/js/year.js`
  - `assets/fonts/*`
  - `assets/icons/*`
  - `assets/images/*`
- Metadata and crawl files:
  - `site.webmanifest`, `robots.txt`, `sitemap.xml`
- Utility scripts:
  - `src/main.py` (the only supported CLI entrypoint)
  - `src/use_cases/render_site.py` (application use-case)
  - `src/use_cases/validate_links.py` (application use-case)
  - `src/use_cases/check_js_types.py` (application use-case)
  - `src/site_renderer/models/` (one class per model file)

## Generation workflow

- For content changes, update `content/site-content.json` and rerun `python3 src/main.py --render-site`.
- For styling changes, update `assets/css/styles.source.css` and rerun `python3 src/main.py --render-site`.
- Keep generated HTML and CSS checked in if the render output changes.
- Prefer `python3 src/main.py --render-site --check` when verifying that generated files are up to date.

## HTML conventions to preserve

- Keep semantic structure: `header`, `nav`, `main`, `section`, `article`, `footer`.
- Keep SEO/social tags on each page:
  - `meta description`, canonical URL, Open Graph, Twitter card tags
- Keep favicon and manifest links in each page head.
- Keep language and direction accurate:
  - `lang="en"`
  - `lang="bn"`
  - `lang="ar" dir="rtl"`
  - `lang="ur" dir="rtl"`

## CSS and typography conventions

- Keep styles centralized in `assets/css/styles.source.css`.
- Let `python3 src/main.py --render-site` regenerate the minified `assets/css/styles.css`.
- Reuse existing classes and visual language before adding new ones.
- Maintain responsive behavior for mobile and desktop.
- Avoid inline styles unless there is a strong reason.
- Preserve the language-based font setup:
  - English: `EB Garamond`
  - Arabic: `Scheherazade New`
  - Urdu: `Noto Nastaliq Urdu`
  - Bangla: `Noto Serif Bengali`

## JavaScript conventions

- Prefer no JS when HTML/CSS can solve it.
- Reuse `assets/js/year.js` for:
  - footer year (`#year`)
  - last refreshed date (`#last-refreshed`)
- Prefer external scripts over duplicated inline scripts.

## Content and localization rules

- When updating shared structure or copy, update the source JSON instead of patching generated pages directly.
- Keep equivalent sections aligned across `en`, `bn`, `ar`, and `ur` unless the difference is intentional.
- For Arabic text, moderate tashkeel is acceptable when it improves readability.
- Use English numerals in Arabic and Urdu localized copy.
- Use Bangla numerals in Bangla localized copy.
- Keep route stability. Do not rename public page files without explicit instruction.

## Validation checklist (after edits)

1. Run `python3 src/main.py --render-site`.
2. Run `python3 src/main.py --render-site --check`.
3. Run `python3 src/main.py --validate-links`.
4. Verify local preview with `python3 -m http.server 8080`.
5. Spot-check at least:
   - one root page
   - one localized RTL page
   - one localized LTR page
6. Ensure no broken asset paths and no removed SEO/meta essentials.

## Out of scope by default

- No migration to SPA frameworks.
- No TypeScript/toolchain setup.
- No package-manager driven refactors.
- No replacement of the current Python render pipeline unless explicitly requested.
- No major redesign of information architecture unless asked.
